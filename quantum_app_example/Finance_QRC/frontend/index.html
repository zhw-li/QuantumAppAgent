算法说明 / Algorithm Description

Quantum Reservoir Computing (QRC)

Quantum Reservoir Computing uses a fixed random quantum circuit as the "reservoir" to project classical input data into a high-dimensional quantum Hilbert space. The quantum reservoir's intrinsic nonlinearity (from quantum gate operations and entanglement) replaces the classical random weight matrix of traditional Echo State Networks.

Key Differences from Classic Reservoir Computing

  • Reservoir: QRC uses a fixed quantum circuit (RY/RZ rotations + CNOT entanglement); Classic RC uses a random sparse recurrent weight matrix.
  • State extraction: QRC measures Pauli-Z expectation values on each qubit (n_qubits-dimensional feature); Classic RC reads the full hidden state vector (n_reservoir-dimensional).
  • Readout: Both use Ridge regression from reservoir states to target output.
  • Training: Only the readout weights are trained; the reservoir (quantum or classical) is frozen after initialization.

Parameter Efficiency Advantage

QRC achieves comparable or better prediction accuracy with orders of magnitude fewer parameters. A 4-qubit QRC circuit uses only ~46 total parameters (20 circuit + 26 readout), while a Classic RC with 100 nodes requires ~9,700 parameters. This makes QRC especially promising for edge deployment and resource-constrained scenarios.
